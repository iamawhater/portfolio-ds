Flagship · LLM Systems

Custom Chatbot Creator

A productized, multi-tenant RAG application that turns any document into a chatbot

Full-stack web app that lets a user upload a document and instantly create their own retrieval-augmented chatbot. The backend chunks and embeds the document, retrieves relevant passages for each question, and grounds the answer in those passages, the standard production defense against LLM hallucination.

The interesting work is not the model call. It is everything around it: per-user session isolation so two users never see each other's data, file-type parsing for .pdf, .docx, and .txt, persistent conversation history in browser-local storage, multi-bot management with delete confirmations, and proper async error handling at every step. Deployed on real infrastructure with a custom domain and HTTPS, not a Streamlit demo.

Computer Vision · Quality

Ceres: Automated Visual Inspection for Grain Quality

CNN-based defect detection across multiple deep-learning architectures

A computer vision pipeline that grades agricultural product as healthy or damaged from images, automating an inspection task that currently takes a human expert 6 to 8 years of training to perform at roughly 80 percent inter-rater accuracy.

Trained and compared multiple architectures (ResNet, MobileNet, R-CNN) in PyTorch over 50 epochs on 2,613 labeled images, and explored SAM (Segment Anything Model) for region-level analysis. Best run reached 95.6 percent overall accuracy and 92.3 percent F1. The write-up shows the train and validation loss diverging rather than hiding it, identifies the damaged class as the honest accuracy ceiling, and scopes a candid data and imaging roadmap rather than overclaiming production-readiness.

Plant Disease Classifier: When 96 Percent Accuracy Lies

A deployed image classifier and a study of what high accuracy hides

A three-class image classifier deployed as a working web app. Reached 96 percent test accuracy via transfer learning on ResNet. The published write-up is the actual deliverable: misclassification analysis and layer-wise feature visualization showing exactly where a high-accuracy model still fails. A pumpkin leaf misclassified because no cucurbit leaves existed anywhere in training. The argument: models should be able to say "I do not know" rather than be confidently wrong.

Multimodal · Foundation Models

DeepSeek Janus-Pro Local Deployment

Running a multimodal foundation model on consumer GPU hardware

Local deployment of DeepSeek's Janus-Pro, a multimodal model that handles both text-to-image generation and image understanding from a single architecture. Set up the full environment on Windows: CUDA toolkit, PyTorch with the right CUDA build, virtual environment, model weights downloaded from Hugging Face, end-to-end inference pipeline.

Hands-on experience with the practical constraints that matter for on-device AI: the 1B parameter model ran on a single 8GB GPU, the 7B model crashed the system. That tradeoff between model size and hardware capacity is the same problem behind quantization, hybrid local-plus-cloud architectures, and edge deployment.

Classical CV · OpenCV

Parking Lot Occupancy Inspection

A classical computer vision pipeline, the lightweight baseline before deep learning

A real-time visual inspection pipeline that classifies parking-spot occupancy from camera images using OpenCV. Manually defined regions of interest, Gaussian smoothing, Canny edge detection, and a per-region edge-density threshold to flag occupied versus empty spots.

Included to show range. Not every CV problem needs a CNN. Knowing when to reach for OpenCV versus PyTorch is the kind of judgment a factory or production team values, because classical pipelines are cheaper to run, easier to debug, and often deployed as the first version before escalating to deep learning.

02 How I Work The part employers actually care about

ADistrust the metric

A high number is a hypothesis, not a conclusion. I score it, then go looking for where it breaks. Misclassifications, held-out edge cases, the input the model never saw.

BKeep the data honest

Preserve the rawest version of the data the analysis can use. Document every decision at every fork, including the ones I overruled and why.

CValidate against known truth

The strongest check on a pipeline is whether it independently recovers something already established. If it can rediscover the known, it's earned the right to ask the unknown.

DShip it

A model in a notebook helps no one. These projects are deployed: full-stack apps, live URLs, real infrastructure. Real tools used through a browser, not screenshots of accuracy curves.
